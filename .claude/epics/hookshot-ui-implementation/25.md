# Task: Project Setup & Infrastructure

## Description

Initialize the Hookshot frontend project with Vite + React + TypeScript, configure all build tools, styling frameworks, and establish the project structure. This is the foundational task that sets up the development environment for all subsequent work.

**Scope:**
- Create new Vite project with React + TypeScript template
- Install and configure Tailwind CSS with dark mode support (pure CSS, no shadcn/ui)
- Configure build pipeline and development tools
- Establish folder structure following best practices
- Configure linting, formatting, and type checking
- Set up custom UI components using pure Tailwind CSS

## Acceptance Criteria

- [ ] Vite project created and runs successfully (`npm run dev`)
- [ ] TypeScript configured in strict mode with zero errors
- [ ] Tailwind CSS installed and configured with custom theme
- [ ] Custom UI components created using pure Tailwind CSS
- [ ] ESLint configured with React + TypeScript rules
- [ ] Prettier configured for consistent formatting
- [ ] Project structure created with all necessary folders
- [ ] Git repository initialized with proper .gitignore
- [ ] Package.json has minimal dependencies (pure CSS approach)
- [ ] Development server starts without errors or warnings

## Technical Details

### Implementation Steps

1. **Initialize Vite Project**
   ```bash
   npm create vite@latest frontend -- --template react-ts
   cd frontend
   npm install
   ```

2. **Install Tailwind CSS**
   Follow the official Vite guide: https://tailwindcss.com/docs/guides/vite

   ```bash
   npm install -D tailwindcss postcss autoprefixer
   npx tailwindcss init -p
   ```

   Configure `tailwind.config.js`:
   ```javascript
   /** @type {import('tailwindcss').Config} */
   export default {
     content: [
       "./index.html",
       "./src/**/*.{js,ts,jsx,tsx}",
     ],
     darkMode: 'class', // Enable dark mode with class strategy
     theme: {
       extend: {
         colors: {
           // Use CSS variables for theming
           background: 'var(--background)',
           surface: 'var(--surface)',
           'surface-hover': 'var(--surface-hover)',
           border: 'var(--border)',
           'text-primary': 'var(--text-primary)',
           'text-secondary': 'var(--text-secondary)',
           'text-tertiary': 'var(--text-tertiary)',
           'accent-blue': 'var(--accent-blue)',
           'accent-green': 'var(--accent-green)',
           'accent-red': 'var(--accent-red)',
           'accent-yellow': 'var(--accent-yellow)',
           'accent-purple': 'var(--accent-purple)',
         },
       },
     },
     plugins: [],
   }
   ```

3. **NO shadcn/ui** - Use pure Tailwind CSS
   Create custom components manually using Tailwind utility classes.
   All UI components will be built from scratch for full control.

4. **Configure TypeScript**
   Update `tsconfig.json`:
   ```json
   {
     "compilerOptions": {
       "strict": true,
       "noUnusedLocals": true,
       "noUnusedParameters": true,
       "noFallthroughCasesInSwitch": true,
       // ... other strict settings
     }
   }
   ```

5. **Install Development Dependencies**
   ```bash
   npm install -D eslint @typescript-eslint/parser @typescript-eslint/eslint-plugin
   npm install -D eslint-plugin-react eslint-plugin-react-hooks
   npm install -D prettier eslint-config-prettier
   npm install lucide-react  # For icons only
   ```

6. **Create Project Structure**
   ```
   src/
   ├── components/
   │   ├── layout/
   │   ├── endpoint/
   │   ├── request/
   │   ├── detail/
   │   └── ui/          # Custom UI components (NOT shadcn/ui)
   ├── hooks/
   ├── lib/
   │   ├── api.ts
   │   ├── websocket.ts
   │   ├── types.ts
   │   └── utils.ts
   ├── App.tsx
   ├── main.tsx
   └── index.css
   ```

7. **Configure Build Scripts**
   Update `package.json`:
   ```json
   {
     "scripts": {
       "dev": "vite",
       "build": "tsc && vite build",
       "preview": "vite preview",
       "lint": "eslint . --ext ts,tsx --report-unused-disable-directives --max-warnings 0",
       "format": "prettier --write \"src/**/*.{ts,tsx,css}\""
     }
   }
   ```

### Key Files to Create

- `frontend/vite.config.ts` - Vite configuration
- `frontend/tailwind.config.js` - Tailwind configuration
- `frontend/tsconfig.json` - TypeScript configuration
- `frontend/.eslintrc.cjs` - ESLint configuration
- `frontend/.prettierrc` - Prettier configuration
- `frontend/src/index.css` - Global styles with Tailwind imports
- `frontend/.env.example` - Environment variables template

### Design Tokens (CSS Variables)

Create in `src/index.css`:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

@layer base {
  :root {
    /* Light mode */
    --background: #ffffff;
    --surface: #f9fafb;
    --surface-hover: #f3f4f6;
    --border: #e5e7eb;
    --text-primary: #111827;
    --text-secondary: #6b7280;
    --text-tertiary: #9ca3af;
    --accent-blue: #3b82f6;
    --accent-green: #10b981;
    --accent-red: #ef4444;
    --accent-yellow: #f59e0b;
    --accent-purple: #8b5cf6;
  }

  .dark {
    /* Dark mode (default) */
    --background: #0a0a0a;
    --surface: #1a1a1a;
    --surface-hover: #2a2a2a;
    --border: #333333;
    --text-primary: #e5e5e5;
    --text-secondary: #a1a1a1;
    --text-tertiary: #6b7280;
    /* Accent colors stay the same for consistency */
  }

  * {
    @apply transition-colors duration-200;
  }
}
```
